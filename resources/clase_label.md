# 🛡️ Diccionario de Clases y Tipos de Ataque

| # | Clase (valor en Label) | Categoría de ataque | Descripción del ataque | Protocolo afectado | Frecuencia aprox. | Dificultad de detección |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | BENIGN | Normal | Tráfico de red legítimo generado por usuarios reales realizando actividades cotidianas (navegación web, email, transferencia de archivos, streaming). Representa la clase mayoritaria del dataset (~80%). | TCP, UDP, HTTP, HTTPS | ~80% del total | Referencia |
| 2 | DoS Hulk | DoS / DDoS | Ataque de Denegación de Servicio usando la herramienta Hulk. Genera una gran cantidad de solicitudes HTTP únicas y aleatorias para saturar el servidor web y consumir sus recursos. | HTTP / TCP | Alta (~10%) | Media — patrones de volumen |
| 3 | PortScan | Reconocimiento | Escaneo de puertos para descubrir qué servicios están activos en un host objetivo. Técnica de reconocimiento previa a un ataque. Genera muchos SYN a diferentes puertos en poco tiempo. | TCP / UDP | Media (~5%) | Media — muchos SYN sin completar |
| 4 | DDoS | DoS / DDoS | Ataque Distribuido de Denegación de Servicio. Múltiples fuentes envían tráfico masivo al objetivo para saturarlo. En CICIDS2017 se simuló usando LOIC (Low Orbit Ion Cannon). | UDP / TCP / HTTP | Media (~5%) | Media — alto volumen desde múltiples IPs |
| 5 | DoS GoldenEye | DoS / DDoS | Ataque DoS con la herramienta GoldenEye. Mantiene conexiones HTTP abiertas enviando cabeceras incompletas para agotar los recursos del servidor web. | HTTP / TCP | Baja (~1%) | Media — conexiones lentas y persistentes |
| 6 | FTP-Patator | Fuerza Bruta | Ataque de fuerza bruta sobre el protocolo FTP usando la herramienta Patator. Intenta múltiples combinaciones de usuario/contraseña para obtener acceso no autorizado. | FTP (TCP puerto 21) | Baja (~1%) | Baja — muchos intentos de login fallidos |
| 7 | SSH-Patator | Fuerza Bruta | Ataque de fuerza bruta sobre el protocolo SSH usando Patator. Similar a FTP-Patator pero dirigido al servicio SSH (administración remota). | SSH (TCP puerto 22) | Baja (~1%) | Baja — muchos intentos de login fallidos |
| 8 | DoS slowloris | DoS / DDoS | Ataque DoS lento (slow attack) que abre muchas conexiones HTTP parciales y las mantiene abiertas enviando cabeceras periódicamente. Agota el pool de conexiones del servidor sin generar mucho tráfico. | HTTP / TCP | Baja (<1%) | Alta — bajo volumen, difícil de distinguir |
| 9 | DoS Slowhttptest | DoS / DDoS | Ataque DoS lento similar a Slowloris pero más versátil. Puede atacar tanto las cabeceras (slow headers) como el cuerpo del mensaje (slow body). Usa la herramienta Slowhttptest. | HTTP / TCP | Baja (<1%) | Alta — bajo volumen, difícil de distinguir |
| 10 | Bot | Botnet | Tráfico generado por bots (software malicioso automatizado) que forma parte de una botnet. Puede incluir comunicación C&C (Command & Control), spam o preparación para otros ataques. | TCP / HTTP / IRC | Muy baja (<0.5%) | Muy alta — imita tráfico normal |
| 11 | Web Attack – Brute Force | Web Attack | Ataque de fuerza bruta sobre aplicaciones web. Intenta múltiples combinaciones de credenciales en formularios de login HTTP. Diferente a FTP/SSH-Patator porque apunta a la capa de aplicación web. | HTTP (TCP puerto 80/443) | Muy baja (<0.3%) | Alta — se mezcla con tráfico HTTP legítimo |
| 12 | Web Attack – XSS | Web Attack | Ataque de Cross-Site Scripting. Inyección de scripts maliciosos en páginas web vistas por otros usuarios. Se detecta por patrones en los parámetros de las solicitudes HTTP. | HTTP | Muy baja (<0.1%) | Muy alta — requiere análisis de payload |
| 13 | Infiltration | Infiltración | Ataque de infiltración que simula a un atacante que ya tiene acceso a la red interna y busca pivotar o exfiltrar datos. Es el ataque más difícil de detectar porque el comportamiento parece legítimo. | Múltiples protocolos | Extremadamente baja (<0.01%) | Extremadamente alta — comportamiento interno |
| 14 | Web Attack – SQL Injection | Web Attack | Ataque de inyección SQL. El atacante inserta código SQL malicioso en campos de formularios web para manipular la base de datos (extraer datos, bypassear autenticación, etc.). | HTTP (TCP puerto 80/443) | Muy baja (<0.05%) | Muy alta — requiere análisis de payload HTTP |
| 15 | Heartbleed | Vulnerabilidad | Explotación de la vulnerabilidad CVE-2014-0160 (Heartbleed) en OpenSSL. Permite leer memoria del servidor enviando paquetes TLS malformados de tipo heartbeat. Una de las vulnerabilidades más graves de la historia. | TLS/SSL (TCP puerto 443) | Extremadamente baja (<0.01%) | Muy alta — poquísimas instancias en el dataset |
